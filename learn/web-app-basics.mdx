Web application testing starts with understanding how the browser and server talk to each other. Most web labs become easier once you can read requests, responses, cookies, parameters, and redirects.

## Building Blocks

- **Request**: What the client sends to the server.
- **Response**: What the server returns.
- **Header**: Metadata attached to a request or response.
- **Cookie**: A small value the browser stores and sends back to the site.
- **Session**: Server-side state that usually connects requests to a logged-in user.
- **Parameter**: User-controlled input sent through the URL, body, or headers.

## Practice Loop

Start with one feature at a time. Change one input, observe the response, and write down what changed. Small tests make web behavior easier to reason about.
